# Components Index

This document provides an inventory of all available components in the Bootstrap Agentic Component Library.

## Component Structure

Each component is located in `/components/<component-name>/` and includes:
- `<component-name>.html` - Ready-to-copy HTML snippet
- `<component-name>.md` - Documentation (purpose, variants, usage rules, accessibility)

## Available Components

### Alert
**Path**: `/components/alert/`
- `alert.html` - Alert component snippet
- `alert.md` - Alert documentation

**Purpose**: Display contextual feedback messages to users.

---

### Button
**Path**: `/components/button/`
- `button.html` - Button component snippet
- `button.md` - Button documentation

**Purpose**: Interactive button elements with various styles and states.

---

### Card
**Path**: `/components/card/`
- `card.html` - Card component snippet
- `card.md` - Card documentation

**Purpose**: Flexible container for displaying content with optional header, body, and footer.

---

### Navbar
**Path**: `/components/navbar/`
- `navbar.html` - Navbar component snippet
- `navbar.md` - Navbar documentation

**Purpose**: Responsive navigation bar with brand, links, and CTA group.

---

### Modal
**Path**: `/components/modal/`
- `modal.html` - Modal component snippet
- `modal.md` - Modal documentation

**Purpose**: Dialog boxes for displaying content, forms, or confirmations.

---

### Table
**Path**: `/components/table/`
- `table.html` - Table component snippet
- `table.md` - Table documentation

**Purpose**: Responsive data tables with striped rows, hover effects, status badges, and action buttons.

---

### Form Components
**Path**: `/components/form/`

#### Form Inputs
- `input.html` - Text input snippet
- `textarea.html` - Textarea snippet
- `select.html` - Select dropdown snippet
- `checkbox.html` - Checkbox snippet
- `form.md` - Form components documentation (covers all form snippets)

**Purpose**: Form input elements with proper labels, validation states, and accessibility.

---

## Usage Notes

- All components are designed to work with Bootstrap 5.3.8 via CDN
- Components follow Bootstrap's utility-first approach
- Each component includes accessibility best practices
- Components can be combined and customized using Bootstrap utility classes
- Always refer to individual component documentation for specific usage rules and variants
